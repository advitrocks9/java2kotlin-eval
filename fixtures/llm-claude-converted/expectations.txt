# hypothesis checks for fixtures/llm-claude-converted -- 15 .kt files
# captured from Claude Sonnet 4.5 over the edge-cases/ java corpus. each
# line is a behavior i actually observed; a future capture (different
# model, different prompt) that violates one of these will fail CI.
#
# format: relpath | tag | yes|no | regex | description
#
# kept distinct from fixtures/edge-converted/expectations.txt because
# claude's output disagrees with static j2k in several places. the
# baseline-diff section in reports/llm-claude.md shows the side-by-side;
# this file pins the LLM side so it can drift independently.

# h.01 anonymous Runnable -- claude lifts to a SAM lambda; static j2k
# keeps the anon-object form.
01_anonymous_runnable.kt | sam_lambda_lifted     | yes | private val hook = Runnable\s*\{ | LLM lifted anon Runnable to SAM lambda (static J2K kept anon-object form)
01_anonymous_runnable.kt | not_anon_object_form  | no  | object\s*:\s*Runnable             | LLM did NOT keep the anon-object form

# h.02 const promotion -- claude promotes everything static-final, including
# the public string AND the `1 + 2` expression. that's the gap ConstValFix.kt
# was patching, gone for free upstream.
02_static_final_constants.kt | const_string_promoted   | yes | const\s+val\s+BASE_PATH | LLM promoted public String static final to const val (J2K skips this)
02_static_final_constants.kt | const_computed_promoted | yes | const\s+val\s+COMPUTED  | LLM promoted compile-time-constant expression (1+2) to const val
02_static_final_constants.kt | array_kept_plain_val    | yes | ^\s*val\s+EXCLUDED\s*:\s*IntArray | array initializer correctly stays plain val (not const-eligible)

# h.06 negative instanceof pattern. kotlin's smart cast is purely lexical
# so the java `if (!(o instanceof Integer i))` shape has to be inverted to
# put the type binding in the right scope. claude does it.
06_instanceof_pattern.kt | smart_cast_inversion | yes | if\s*\(o\s*!is\s+Int\) | LLM inverted negative instanceof to !is to fit smart-cast scope

# h.07 try-with-resources. both converters do the .use {} thing. claude
# also drops the over-conservative nullable params -- the source had
# non-null parameters; static j2k marks them String? out of caution.
07_try_with_resources.kt | use_block             | yes | \.use\s*\{                    | try-with-resources -> .use{}
07_try_with_resources.kt | not_overconservative_nullable | no  | fun readOne\(path: String\?\) | LLM did NOT mark non-null param as nullable

# h.08 utility class. claude uses top-level `object Sample` here (same as
# static j2k). interesting that on 02_static_final_constants.kt claude
# wraps in a companion -- so it distinguishes constants-only classes
# (companion) from pure-function utility classes (top-level object). j2k
# doesn't make that distinction.
08_static_utility_class.kt | top_level_object_for_pure_utility | yes | ^object\s+Sample\s*\{ | LLM agrees with J2K on top-level object for pure-utility class
08_static_utility_class.kt | not_companion_for_pure_utility    | no  | companion\s+object    | LLM does NOT add a companion wrapper for the pure-function case

# h.12 overloads-to-defaults. claude does NOT collapse three overloads into
# one fun with default args -- same conservatism as static j2k. so my
# hypothesis (h.12) doesn't land for either.
12_overloaded_default_args.kt | overloads_kept_separate | yes | fun render\(s: String\): String | LLM kept 1-arg overload as a separate fun (no default-arg collapse)
12_overloaded_default_args.kt | three_overloads_present | yes | fun render\(s: String, prefix: String, upper: Boolean\) | full 3-arg render still present
